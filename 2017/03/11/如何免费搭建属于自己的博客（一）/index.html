如何免费搭建属于自己的博客（一）

发表于 2017-03-11 | 分类于 总结 | | 阅读次数

0. 写在前面

学习Python已有一段时间，特搭建博客，记录成长，分享收获，砥砺前行。作为一个小白，搭建此博客共用时10天左右，中间遇到很多没见过的问题，但都查资料解决。总结共计三篇，第一篇主要介绍如何搭建博客，第二篇介绍如何美化博客并增加功能，第三篇主要介绍如何发布新的博客。

本文配图较少（后悔没截图），若有不明之处请留言，我会尽量解答。

系统为WIN7旗舰版32位，配置如下：

博客基于hexo框架搭建（可自行百度），站点挂载在github，上。

1. 准备工作

以下软件可直接百度安装，推荐用迅雷下载。

  • 安装编辑器：sublime text（用于编辑配置文件）

  • 下载控制台：cmder（强力推荐，作用相当于cmd，无需安装，超级好用）

  • 下载配置环境所需软件：Node.js、git（下载后直接安装）

  • 注册并登录github，右上角选择 Create a new repository，Repository name 填自己的Username，这就是你博客的域名了。注意：Repository name要与你注册用户名相同。

到此，前期准备工作完成。

2. 环境搭建

安装Node.js、git：直接next。
git教程可以参考廖雪峰老师的教程：git教程

安装完成后，在开始菜单里找到“Git”->“Git Bash”，输入以下命令行： 

git config --global user.name '你的名称Username'

回车

git config --global user.email 'Github的邮箱'

回车

安装 Hexo:输入以下命令

npm install -g hexo-cli

时间略长，等几分钟。至此，环境搭建完毕。

3. 生成静态页面

先了解一些hexo的一些常用命令：

hexo new "postName" #新建文章
new page "pageName" #新建页面
hexo generate #生成静态页面至public目录，也可写成hexo g
hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server），也可写成hexo s
hexo deploy #将.deploy目录部署到GitHub
hexo help  # 查看帮助
hexo version  #查看Hexo的版本

在D盘（除C盘外其他盘都可以）建立空文件夹‘test’（随便什么名）。

打开cmder,输入命令转到test文件夹下之后，输入以下命令(创建博客的文件)：

hexo init blog

若出现以下提示，表明成功。

INFO  Start blogging with Hexo!

现在可以启动服务器进行本地预览，键入以下命令：

hexo g  #生成静态页面

hexo s  #启动服务器

若出现以下提示，表明成功。

INFO  Start processing
INFO  Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.

打开浏览器，访问http://localhost:4000/，若有安装“福昕阅读器”相关软件，4000的端口被占用，请输入以下命令，改用5000的端口访问：

hexo s -p 5000

网站初始模样如下（自带一篇hello world文章，可删除）：

恭喜你，离成功只有一步之遥了！

4.配置并发布博客

现在看到的网页为本地网页，若想在互联网上看到，我们还要将网页发布到Github上。

打开cmder，输入以下命令，获取密钥：

ssh-keygen -t rsa -C "Github的注册邮箱地址"

回车，回车，回车，看到

Your public key has been saved in /c/Users/user/.ssh/id_rsa.pub.

在c/Users/user/.ssh中找到id_rsa.pub文件（公钥），使用sublime打开，复制所有内容，打开github的SSH keys（点击传送门直接到达），找到SSH keys，点击New SSH key→Title：blog→Key：粘贴复制内容→ Add SSH key。成功后如下:

在blog目录下，用sublime打开_config.yml文件，修改参数信息，各参数作用如下：

# Hexo Configuration
## Docs: http://hexo.io/docs/configuration.html
## Source: https://github.com/hexojs/hexo/
# Site #站点信息
title:  #标题
subtitle:  #副标题
description:  #站点描述，给搜索引擎看的
author:  #作者
email:  #电子邮箱
language: zh-CN #语言
# URL #链接格式
url:  #网址
root: / #根目录
permalink: :year/:month/:day/:title/ #文章的链接格式
tag_dir: tags #标签目录
archive_dir: archives #存档目录
category_dir: categories #分类目录
code_dir: downloads/code
permalink_defaults:
# Directory #目录
source_dir: source #源文件目录
public_dir: public #生成的网页文件目录
# Writing #写作
new_post_name: :title.md #新文章标题
default_layout: post #默认的模板，包括 post、page、photo、draft（文章、页面、照片、草稿）
titlecase: false #标题转换成大写
external_link: true #在新选项卡中打开连接
filename_case: 0
render_drafts: false
post_asset_folder: false
relative_link: false
highlight: #语法高亮
  enable: true #是否启用
  line_number: true #显示行号
  tab_replace:
# Category & Tag #分类和标签
default_category: uncategorized #默认分类
category_map:
tag_map:
# Archives
2: 开启分页
1: 禁用分页
0: 全部禁用
archive: 2
category: 2
tag: 2
# Server #本地服务器
port: 4000 #端口号
server_ip: localhost #IP 地址
logger: false
logger_format: dev
# Date / Time format #日期时间格式
date_format: YYYY-MM-DD #参考http://momentjs.com/docs/#/displaying/format/
time_format: H:mm:ss
# Pagination #分页
per_page: 10 #每页文章数，设置成 0 禁用分页
pagination_dir: page
# Disqus #Disqus评论，替换为多说
disqus_shortname:
# Extensions #拓展插件
theme: landscape-plus #主题
exclude_generator:
plugins: #插件，例如生成 RSS 和站点地图的
- hexo-generator-feed
- hexo-generator-sitemap
# Deployment #部署，将 lmintlcx 改成用户名
deploy:
  type: git
  repo: 刚刚github创库地址.git
  branch: master

只修改以下参数，每个参数“：”后，都要加一个空格：

修改网站相关信息：

title: xxlee'Study Notes
subtitle: 无他，唯手熟尔。
description: Python/English
author: xxlee2009
language: zh-Hans
timezone: Asia/Shanghai

配置部署（我的是xxlee2009，修改成你自己的）

deploy:
  type: git
  repo: git@github.com:xxlee2009/xxlee2009.github.io.git
  branch: master

至此，配置文件修改完毕，下面我们将博客挂载在Github上。打开cmder，将输入命令转到blog文件夹下：

hexo clean #清楚缓存文件
INFO  Deleted database.
INFO  Deleted public folder.

之后输入，发布到网上：

hexo deploy

然后就可以看到博客发布到网上了。

当然，这只是博客的雏形，想要更多功能，更好看的界面，还需要添加新的插件，使用美观的主题，我将在下一篇进行总结。
